Hugo + Obsidian 目录设计说明

目标

这套目录设计的目标是把 Obsidian 的笔记管理能力和 Hugo 的静态站点生成能力解耦开来：

  • Obsidian 负责管理整个笔记仓库
  • Hugo 只负责把博客相关的 Markdown 转成静态网页
  • GitHub Pages 只负责发布 Hugo 生成的网站

这样做的好处是，博客内容和其他私人笔记、项目笔记、日记不会混在一起，同时 Hugo 站点的职责也保持单一。

设计原则

目录设计遵循下面几条原则：

  • notes/ 是 Obsidian Vault 根目录，也是日常使用的主目录
  • blog/ 是公开博客内容源，只放准备发布的内容
  • site/ 是 Hugo 工程目录，只负责构建和发布
  • 非博客内容默认不进入 Hugo 发布链路
  • 第一版用相对路径软链接连接 blog/ 和 site/

推荐目录结构

notes/
  .obsidian/
  inbox/
  daily/
  knowledge/
  project/
  blog/
    content/
      posts/
      pages/
    assets/
      images/
  site/
    content/
      posts -> ../../blog/content/posts
      pages -> ../../blog/content/pages
    static/
      images -> ../../blog/assets/images
    .github/
      workflows/
        hugo.yml
    hugo.toml

各目录职责

notes/

这是 Obsidian 的 Vault 根目录。你在 Obsidian 里应该直接打开这个目录，而不是只打开 blog/ 或 site/。

初级阶段学习计划

1. 阶段定位

适用时长：第 1-12 周

适用对象：

  • 刚开始接触冥想
  • 很难稳定坐下
  • 经常走神、烦躁、困倦
  • 不清楚“怎么练才算对”

这个阶段的核心，不是追求平静、空白、轻盈感，而是建立最基础的训练回路：

注意偏离 -> 觉察到偏离 -> 轻轻回来

2. 阶段目标

  • 建立稳定的练习习惯
  • 找到一个固定的注意锚点
  • 学会处理走神、困倦、烦躁
  • 初步建立身体觉察
  • 不再把“走神”当成失败

3. 训练主线

本阶段主线：

呼吸专注 -> 简短身体扫描 -> 少量开放觉察预备

比例建议：

  • 呼吸专注：70%
  • 身体扫描：25%
  • 开放觉察预备：5%

4. 12 周周计划

第 1 周

重点：先建立出现频率，不要求状态好。

每日内容：

  • 呼吸专注 5 分钟
  • 练习结束后静坐 1 分钟

要求：

  • 每周 5-7 天
  • 固定一个时间段
  • 固定一个练习位置

第 2 周

重点：让“坐下开始练”变得更自然。

每日内容：

  • 呼吸专注 5-8 分钟
  • 练习结束后静坐 1 分钟

要求：

  • 每周 5-7 天
  • 继续固定时间与地点
  • 呼吸位置只选一个，例如鼻尖、胸口或腹部

第 3 周

重点：从能坐下，进入能发现走神。

高级阶段学习计划

1. 阶段定位

适用时长：1 年以上

高级阶段不等于神秘体验更多，也不等于练得更久。它更看重三件事：

  • 可迁移性
  • 一致性

2. 阶段目标

  • 长期维持正式练习
  • 深化开放觉察与慈悲训练
  • 在压力、冲突、疲劳、决策中保持觉察
  • 让生活本身成为主要训练场
  • 逐渐形成更稳定的平衡心

本阶段主线：

专注稳定维护 -> 开放觉察深化 -> 慈悲与平等心 -> 高密度日常整合

  • 冲突或压力来时主动暂停若干次

每次只需 10 秒到 3 分钟

冥想学习指南：从入门到进阶与成熟

一、引言

1.1 为什么要学习冥想

冥想并不是某种神秘修行，而是一类可以被系统学习和反复训练的心智练习。美国国家补充与整合健康中心（NCCIH）在官方综述中指出，冥想与正念相关练习对压力、焦虑和抑郁症状可能有帮助，但效果会受到练习方式、课程质量、个体差异和使用场景影响。[1]

从神经科学视角看，部分研究发现，长期冥想练习与大脑功能和连接模式的变化有关，例如默认模式网络（DMN）活动差异、情绪加工相关脑区反应变化，以及注意监测和自我调节相关机制的改变。[2][3][4][5] 这些发现并不意味着冥想可以被神化成万能方法，但足以说明：它不是空泛口号，而是一种有实证基础的训练路径。

更重要的是，冥想并不只发生在打坐时。它真正的价值，通常体现在生活中是否更容易察觉自动反应，是否能在压力、冲突、疲劳和情绪波动中保留一点观察和选择的空间。[5][6]

1.2 冥想常见误解

在开始之前，有必要先澄清几个常见误解。

误解一：“冥想就是把脑子练空。”
不是。冥想的核心不是压制念头，而是提升对念头、身体感觉和情绪活动的觉察能力。注意力跑掉后，能够看见它并回来，这个过程本身就是训练内容。[2]

误解二：“冥想必须盘腿坐很久。”
不是。标准化正念课程 MBSR 包含坐姿练习、身体扫描、行走练习和温和伸展，不要求固定采用某一种姿势。姿势服务于练习，不是练习本身。[7]

误解三：“冥想是万能疗法。”
不是。NCCIH 明确提醒，冥想不能替代必要的医疗或心理治疗；同时，也不是所有人都只会获得正面体验，部分练习者可能出现焦虑加重、痛苦记忆浮现或其他不适。[1][8][9]

误解四：“我老是走神，说明我不适合冥想。”
不是。走神不是失败。研究与实修框架都强调，发现走神并把注意力带回来，本身就是入门阶段的核心训练。[2][10]

1.3 本文的结构与阅读方式

本文先说明冥想的定义、类型和核心能力，再讨论其作用、机制与边界，然后给出三套常见阶段框架，最后进入实操方法、常见困难与资料建议。

如果你是初学者，建议按顺序阅读。先建立概念，再进入方法，会更容易避免把冥想误解成“追求特殊体验”的活动。

二、概念基础

2.1 冥想的定义

现代心理学与认知科学视角

在现代研究语境中，冥想通常被理解为一组通过训练注意力、觉察和心理调节过程来影响体验与行为的练习。Lutz、Slagter、Dunne 和 Davidson 在经典综述中指出，冥想的核心机制之一是对注意的调节与监测，即学习如何维持注意、发现偏离，并在不自动卷入的情况下重新定向。[2]

美国心理学会（APA）的科普页面则把正念冥想概括为一种帮助人们以更当下、更少评判的方式接触经验的练习路径。[11]

东方传统修习语境

在东方传统中，冥想通常被理解为对“心如何运作”的训练。其重点不只是放松，而是通过长期练习培养稳定的注意力、清晰的觉察力以及较少被情绪和执着牵引的能力。传统中的“止”偏向专注与安定，“观”偏向洞察与如实觉察。

无著菩萨在《大乘阿毗达磨集论》中提出的“九住心”，是东方传统中较具代表性的注意力渐进训练框架之一。现代一些实修体系，尤其是十阶段模型，会把它作为古典依据之一。[10]

冥想与正念（Mindfulness）的关系与区别

“冥想”是更大的上位概念，包含多种练习方式；“正念”既可以指一种心理品质，也可以指一类具体练习。卡巴金将正念定义为：有意识地、在当下、并以不评判的方式关注经验。[12]

因此，更准确的说法是：正念是冥想中的重要分支，但并不等于全部冥想。专注训练、开放觉察训练、慈心与慈悲训练都可能与正念有关，但它们的直接训练目标并不完全相同。[2][6]

2.2 冥想的主要类型

依据当前研究中最常见的分类，冥想通常可分为三大主要类型：[2][6]

专注注意（Focused Attention, FA）

把注意力放在单一对象上，例如呼吸、腹部起伏、鼻尖触感或脚步。当注意力偏离时，察觉，再回来。它主要训练注意维持、冲突监测和分心后的回收能力，是最典型的入门路径。[2]

开放监测（Open Monitoring, OM）

不再把注意力死守在一个点上，而是对呼吸、念头、声音、情绪和身体感觉保持开放、非反应式的觉察。它要求练习者既清醒又不过度抓取，因此通常建立在一定专注力基础上。[6]

慈心 / 慈悲冥想（Loving-kindness / Compassion）

通过有意识地培养对自己和他人的善意、温和、关怀与不敌对反应，来改变情绪和关系模式。它在现代研究里常与 FA、OM 并列为三大主要冥想类型之一。[6]

正念减压（MBSR）体系

MBSR 由 Jon Kabat-Zinn 于 1979 年在马萨诸塞大学医学院创立，是当前研究和应用最广的标准化冥想课程之一。典型结构为 8 周，课程中通常包含坐姿冥想、身体扫描、正念行走和温和伸展。它不是单一技巧，而是把多种练习整合进一个可复制的训练框架。[7][12]

一、为什么网络上的气功资料不可轻信

网络平台（抖音、小红书、百度、快手等）的气功内容存在以下结构性问题，导致大量学习者练偏、练错：

  1. 缺乏系统理论：视频以"动作教学"为主，缺少对三调（调身、调息、调心）整体框架的讲解，使练习者只学到"形"，遗失了"意"与"息"。
  2. 功法选择无依据：未考虑练习者的年龄、体质、健康状况、心理特质，一刀切地推荐"高阶功法"或"神奇功法"。
  3. 商业动机干扰：大量"气功大师"“丹道秘传"内容夹杂敛财与造神成分，国家体育总局在2020年公告中已明确提示此类风险。
  4. 缺乏反馈机制：线上自学无法纠偏，出现偏差无人知晓。

二、气功的定义与核心理论：三调

什么是气功

气功是中国传统的身心锻炼方法，通过**调身（形体调整）、调息（呼吸引导）、调心（意识调控）**三个层次的协调配合，达到养生保健、防病治病的目的。

来源：中国人民大学出版社等权威出版物，以及 people.com.cn 有关气功基础理论的报道，来源可靠。

三调的具体内涵

层次 核心内容 要点
调身 对姿势、动作和形体的调整 “形正气顺”；松腰、沉胯、虚灵顶劲；形不正则气不顺
调息 对呼吸的自觉调控 要求"深、长、细、匀、轻”；腹式呼吸为常见方式；是沟通调身与调心的桥梁
调心 对意识、思维、情绪的调控 排除杂念、达到"入静"；可采用"意守丹田"等方法；三调中的核心环节

1. 虚领顶劲

重点

  • 头顶（百会穴）向上轻轻领起
  • 下巴微收，不是低头，是"收"
  • 脖子不僵硬，保持弹性

目标

拉开颈椎间隙，让脊柱有向上的"弹性空间"。头领起来，整条脊柱才能挂得住，不会塌。

大白话

想象有人用一根绳子拴住你头顶的头发，轻轻往上提。身体被"吊"起来，但不是绷直——是悬吊，像木偶被线提着，身体自然垂下来。

❌ 常见错误

错误 后果
故意仰头挺脖子 颈椎受压，气血不通
低头看地 整条脊柱前倾塌掉
脖子僵硬，像军训立正 肩颈紧张，越站越累

✅ 自检方法

站好后让人从侧面看：耳朵、肩膀、胯骨应大致在一条垂线上。如果耳朵跑到肩膀前面了，说明头没领好。

2. 含胸拔背

重点

  • “含"不是"扣”，胸口微微内含，不是弓背驼背
  • “拔"是脊柱向上拉长，背部有展开的感觉
  • 两肩胛骨微微外展，不夹紧

目标

让胸腔松沉，横膈膜放松下降；背部筋膜展开，和手臂形成连接。

大白话

想象你穿了一件湿透的T恤贴在后背上。后背自然展开、平铺——这就是"拔背”。胸口不是挺出去的，是"自然收回来一点"，像叹了口气之后胸口的状态。

❌ 常见错误

错误 后果
故意弓背驼背 脊柱弯曲，气息压在胸口
挺胸抬头（军姿） 腰椎代偿，后腰僵硬
两肩胛骨使劲往中间夹 背部筋膜打不开，和手臂断连

✅ 自检方法

站好后注意胸口：如果感觉胸口"空空的、松松的"，呼吸自然往下走，就对了。如果感觉胸口憋着一口气，就是含胸过度或挺胸了。

第一层：结构

最容易理解。

  • 头顶向上

  • 尾闾向下

身体撑圆。

所谓松只是塌。

第二层：筋膜张力

现代运动科学把这个称为：

Tensegrity

人体并不是骨头堆起来的。

而是：

  • 骨骼负责支撑

混元桩气功练习复盘与经验指南

日期： 2026年5月27日

阶段： 新手筑基期

📋 核心复盘：今日问题与真相拆解

问题 1：用 100% 意念（臆想）去牵动和护送体内热流走“小周天”

  • 身体信号： 浑身燥热，站完桩后感觉非常耗费精气神。

  • 真相拆解：

    • 犯了“意念过重（火重）”的忌讳。现阶段体内的“气”属于微弱的“大药不足”，强行用高度集中的意念去搬运它，相当于“烤干锅”，极度消耗心血和脑力。

    • 真通小周天是“水到渠成、气足自通”的自然现象。 应该像蓄水池一样先在丹田“攒钱”，等能量满了自己去冲关，而不是用大脑去强行“画圈”。

问题 2：头顶（大脑）明显发热，但腰部以下冰凉不热

  • 身体信号： 气血上涌，头部燥热。

  • 真相拆解：

    • 属于典型的“虚火上炎（上实下虚）”。健康的站桩状态应该是“头凉脚热”。

    • 根源在于扫描顺序和比例不对： 每次扫描都从头部开始，且在头部停留、动脑的比例过高。“意到气到”，导致全身气血被强行“抽”到头部。此时下半身肌肉僵硬，经络像水管一样被踩住，气血无法下行。

问题 3：练习到后半段腿一直在抖

  • 身体信号： 肌肉无法控制地剧烈颤抖。

  • 真相拆解：

    • 这是新手的“换力期”。浅层的大块肌肉疲劳，身体正在强迫深层的骨骼肌和筋膜接管体重。

    • 另一个原因是气血卡在头上，下半身缺乏气血滋养，肌肉在“缺血”状态下干活产生生理颤抖。

🚨 紧急重大纠偏：关于“发热部位”的误判

📝 经验笔记更新

  • 旧误判： 以为是“腹部/下丹田”发热。

  • 新核实： 实际发热部位在两个乳头中间偏上的位置（膻中穴）。

  • 纠偏结论： 此现象并非“下丹田发热”，而是典型的**“气聚膻中”与“心火上炎（虚火）”**。

1. 为什么热量会聚在两乳之间（膻中穴）？

  • 呼吸的“隐形用力”把气憋在了胸腔： 由于花了 90% 的意识在关注呼吸，虽然主观追求自然，但大脑给的注意力太重，导致胸腔和横膈膜在潜意识里处于半紧张、半憋气状态。气血顺着呼吸往下走，刚走到胸口就被紧张的横膈膜挡住，无法下行至小腹，从而全部堆记在“上气海”（膻中穴），导致局部发热。

  • “保安”离犯罪现场太近： 精力过度集中在胸口的呼吸上，意念在胸口，气血就聚在胸口。火聚在胸口，自然就会往上熏头部，导致头顶跟着一起发热。

2. 气聚膻中的隐患

传统的下丹田（小腹）发热是温和、养人的；而中丹田（胸口）发热，对新手而言往往是燥热、伤神的。长期如此会出现：

  • 胸闷、憋气： 感觉胸口压着石头，呼吸不顺。

  • 心慌、心跳加快： 热量离心脏太近，容易导致心火亢盛。

  • 脾气暴躁、焦虑： 心火上浮，人容易急躁、静不下心。

🛠️ 核心心法升级：如何当一个“冷酷的保安”

站桩不是“什么都不想”（容易睡觉昏沉），也不是“拼命死想”（容易杂念散乱），而是要寻找第三种状态：保持 100% 的清醒，但身体和思维不参与劳动。

  • 保安看监控： 坐在监控室（意识清醒），看着屏幕（呼吸进出、大腿酸痛、热流移动）。你知道它们在发生，但你不用冲进屏幕里去干预。

  • 背景噪音法则： 把大腿的酸、累、抖，当成高级餐厅里的背景音乐，或者窗外的下雨声。允许它存在，但不跟它较劲，带着这个噪音继续站你的桩。

量化交易平台 v3.0 架构评估

总体印象

这份文档是一份相当成熟的架构设计，v3.0 相比 v2.0 有显著改进，特别是时间模型、OMS状态机、资金会计的引入。但仍存在若干系统性问题值得深入讨论。

优点

分层设计清晰。用户入口层 → 平台核心层 → 数据层的三层分离符合量化平台的最佳实践，核心概念（时钟、撮合、账户）有明确的抽象接口。

阶段化落地务实。P1/P2/P3 的拆分避免了过度工程化，Phase 1 聚焦"单机、单策略、稳定跑通"是正确的优先级判断。

OMS 状态机设计正确。订单状态的合法/非法转换图完备，ExecutionReport 的引入统一了状态变更的通知模型，这是 v2.0 最大的进步之一。

VectorStrategy “弱状态"修正。将纯无状态改为携带计算状态 generate_signals(bars, state) → (SignalArray, state) 是务实的，避免了在线特征无法缓存的问题。

问题与优化建议

🔴 严重问题（会导致系统在实盘中出错）

问题一：StrategyContext 持有 IBroker 的安全隐患

文档中 StrategyContext.broker: IBroker 允许策略直接调用 broker.submit_order()，绕过风控链。

# 危险：策略可以绕过风控直接下单
class MyStrategy(EventStrategy):
    def on_bar(self, bar, ctx):
        ctx.broker.submit_order(Order(...))  # 跳过了 risk_chain.check()
        ctx.buy(...)                          # 走正常风控路径

建议：Context 只暴露 buy/sell/close 便捷方法，内部由引擎统一走风控链，彻底移除 context.broker 的直接引用，或者改成只读的 context.broker_status（仅查询，不可操作）。

问题二：时间模型缺少"数据截断"的强制机制

文档声明 DataView 会"自动截断到 now()"，但没有定义如何强制实施。

# 策略可能轻易拿到未来数据
class MyStrategy(EventStrategy):
    def on_bar(self, bar, ctx):
        # history() 内部是否真的被截断？
        # 如果策略直接访问 DataManager 而非 ctx.history()，怎么办？
        future_data = self.data_manager.get_bars("SH.600000", end="2025-01-01")

未来函数（Look-ahead bias）是回测失效的最常见原因，文档只在 StrategyContext.history() 层面提到截断，但没有在架构层面做硬性隔离。
